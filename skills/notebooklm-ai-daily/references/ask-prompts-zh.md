# NotebookLM Ask 模板（中文）

## A. 筛选提示词（强制返回 5 条）

将以下占位符替换后使用：
- `{NOW}`：当前时间（例如 `2026-02-12 11:35`）
- `{WINDOW_START}`：开始时间（例如 `2026-02-11 00:00`）
- `{WINDOW_END}`：结束时间（例如 `2026-02-12 23:59`）

```text
当前时间是 {NOW}。请严格只基于本笔记本已收录来源，在时间范围 {WINDOW_START} 到 {WINDOW_END} 内筛选资讯。

筛选范围：
1) 新模型发布
2) 新 Agent
3) 实用工具 / Skill（如 remotion、pencil 这类能直接提升生产力的工具）
4) AIGC（图像/视频/音频/多模态生成）

硬性排除：
- 向量模型、Embedding、向量数据库、检索基础设施类更新
- 无明确发布时间
- 纯观点、二次搬运、营销软文

筛选规则：
- 同一事件只保留一个主来源，优先官方或一手来源
- 结果必须恰好 5 条，不多不少
- 若有效资讯不足 5 条，仍输出 JSON 数组，并在每条 reason 中解释为何入选与数量不足原因

输出格式（只输出 JSON，不要任何额外文字）：
[
  {
    "category": "model|agent|tool|aigc",
    "title": "标题",
    "publish_time": "YYYY-MM-DD HH:mm (若来源仅有日期可写 YYYY-MM-DD)",
    "source_name": "来源站点/机构",
    "source_url": "https://...",
    "one_line_value": "一句话价值说明",
    "reason": "入选理由（含业务价值或传播价值）"
  }
]
```

## B. 视频文案提示词（基于上一步 5 条）

> 要求在同一 `conversation_id` 继续提问。

```text
基于你刚刚筛选出的 5 条资讯，生成一份可直接用于 90-120 秒中文介绍视频的口播文案。

要求：
- 结构：开场（10秒）→ 5条快讯主体（每条12-18秒）→ 结尾总结（10秒）
- 风格：信息密度高、客观、少形容词、适合 AI 圈日报
- 每条快讯必须带：事件名、时间、来源机构、对从业者的直接意义
- 文案后追加一个“镜头/画面建议”小节，给出每段对应画面关键词（方便 remotion 制作）
- 输出语言：简体中文
```
