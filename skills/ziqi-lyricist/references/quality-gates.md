## 质量门禁清单（不通过不得交付）

### A. 准备阶段

- [ ] 完成情感四问并给出核心情感
- [ ] 输出 8 类禁用清单并告知用户
- [ ] 主题与词眼来自用户输入
- [ ] 已完成意象检索与风格筛选
- [ ] 手法选择基于意象素材，不是凭空选择

### B. 结构一致性（最高优先级）

- [ ] 段落数量与分析报告一致
- [ ] 段落顺序与分析报告一致
- [ ] 每段行数与分析报告一致
- [ ] 没有自行增加段落
- [ ] 没有自行删除段落

> 结构项任意不通过，整稿作废，必须重写。

### C. 黑名单与反AI痕迹

- [ ] 无时间类高频意象
- [ ] 无空间类高频意象
- [ ] 无动作类高频套路
- [ ] 无感官类空泛描写
- [ ] 无“说不清道不明”等套话
- [ ] 无观察者视角“我偷偷观察你”
- [ ] 避开原曲核心意象与关键短语

### D. 词眼与金句

- [ ] 词眼长度 3-6 字
- [ ] 词眼出现在 Chorus / Final Chorus
- [ ] 金句短促有留白（3-6字优先）
- [ ] 金句不是口号/鸡汤式表达
- [ ] 金句具备情感冲击力

### E. 标签与格式

- [ ] 使用 Suno 方括号标签格式
- [ ] Pre-Chorus 使用 `[Pre-Chorus - Gradual Build Up]`
- [ ] Chorus 使用 `[Chorus - Powerful & Emotional]`
- [ ] Chorus 超过 3 句时插入 `[Keep Power]`
- [ ] Final Chorus 使用 `[Final Chorus - Soaring High]`
- [ ] 无 LRC 时间戳残留（如 `[00:15.50]`）
- [ ] 无 Markdown 标题残留（如 `## Verse 1`）

### F. 主题集中度

- [ ] 所有句子都服务主题
- [ ] 无与主题无关的散点内容
- [ ] 情绪推进路径清晰
- [ ] 有叙事锚点（不是纯抽象）
- [ ] 有感官/场景细节（不是纯概念）

### G. 风险处理

- [ ] 有不确定项时明确标注“待确认”
- [ ] 不虚构不存在的段落与约束
- [ ] 不通过时先修订再展示
