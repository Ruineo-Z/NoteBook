# 企业级 Agent 工程师 6–9 个月成长路线图

---

## 阶段 0：现状盘点（第 0 周）
**目标**：明确基础能力和资源  
**能力/动作**：
- 梳理 Python / AI 编程能力  
- 总结已完成 Agent 项目  
- 列出现有 LLM、RAG 框架、工具链清单  
**产出**：
- 技术现状报告（可自用或面试展示）  

---

## 阶段 1：上下文管理 & 执行链路打基础（第 1–2 个月）
**目标**：掌握企业级 Agent 核心数据流和决策闭环  
**模块**：上下文管理、模型执行链路  
**能力/练习**：
- 搭建多轮 Agent demo，支持短期 session 状态  
- 设计任务状态管理结构（短期/中期/长期）  
- 决策链路演练：LLM 生成 → 工具执行 → 状态更新  
**产出**：
- 状态流转图 + 执行链路图  
- 可运行 Agent Demo  

---

## 阶段 2：观测性与调试能力（第 3–4 个月）
**目标**：让不可控的 Agent 可观测、可调试  
**模块**：观测性、调试性  
**能力/练习**：
- 设计日志 Schema：记录输入、Prompt、上下文、工具调用、输出  
- 实现回放接口，复现单次任务  
- 指标化统计：成功率、fallback 率  
**产出**：
- 调试日志示例  
- 回放任务演示  
- 指标统计报告  

---

## 阶段 3：回滚机制（第 4–5 个月）
**目标**：系统具备安全兜底能力  
**模块**：回滚机制  
**能力/练习**：
- 为 Agent 任务设计自动回滚 + 人工介入节点  
- 模拟失败场景，验证回滚策略  
- 插入事务节点保证状态一致性  
**产出**：
- 回滚流程图  
- 回滚演示案例  

---

## 阶段 4：迭代能力（第 5–6 个月）
**目标**：系统可持续优化  
**模块**：是否可迭代  
**能力/练习**：
- 设计指标闭环：任务成功率、错误类型统计  
- 实现 Feedback Loop：日志 → 改进 Prompt / 流程 / 规则  
- 模拟模型升级、流程优化的灰度测试  
**产出**：
- 迭代改进报告  
- 前后对比指标图  

---

## 阶段 5：综合企业级 Agent 项目（第 7–9 个月）
**目标**：整合前四个阶段能力，完成可交付企业级 Agent  
**模块**：全部五个模块  
**能力/练习**：
- 选定一个真实场景（工单处理 / 审批 / 内部知识问答等）  
- 完整设计系统：上下文管理 + 决策链路 + 观测调试 + 回滚 + 迭代  
- 验证可用性、可维护性、可扩展性  
**产出**：
- 完整系统原型（可运行 + 可演示）  
- 面试可讲故事：架构设计 + 技术挑战 + 迭代成果  

---

## 规划特点
1. **模块驱动**：每个月集中练一个核心能力  
2. **产出导向**：每阶段都有可展示的成果  
3. **企业视角**：全部围绕“可交付、可维护、可迭代的系统”  
4. **渐进升级**：从状态管理、执行链路 → 调试 → 回滚 → 迭代 → 综合项目
