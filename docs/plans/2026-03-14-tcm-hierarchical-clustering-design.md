# 中医证候层次聚类脚本设计

## 目标

基于用户提供的 Excel 数据表，按既定统计学方法实现可重复运行的层次聚类分析脚本，并输出最优聚类数、树状图、患者聚类标签及各证候群症状明细。

## 输入

- Excel 文件：`/Users/ruinow/Documents/中医数据_结合西医特征调优版（ai层次聚类法）.xlsx`
- 默认使用首个工作表
- 默认排除基础信息列：`姓名`、`年龄`

## 数据处理规则

1. 对除基础信息外的证候列做二值化校验，要求取值仅包含 `0/1` 或可安全映射到 `0/1` 的布尔值。
2. 空值默认补为 `0`，并在结果汇总中记录补值数量。
3. 计算每个症状的全局发生率，按 `< 5%` 阈值剔除低频症状。
4. 若剔除后特征数不足以支持聚类，则直接报错并停止。

## 聚类方法

- 方法：凝聚型层次聚类
- 距离：欧氏距离，对应 Ward 法内部最小方差目标
- 合并准则：`ward`
- 候选聚类数：`K=4~7`
- 评价指标：
  - 轮廓系数（Silhouette Score）
  - Calinski-Harabasz 指数
- 选优规则：先比较轮廓系数，若并列再比较 C-H 指数

## 输出

输出目录默认设为 `outputs/tcm_clustering/<时间戳>/`，包含：

- `cluster_summary.xlsx`
  - `summary`：输入参数、样本数、保留/剔除特征数、最优 K、缺失值处理情况
  - `k_metrics`：`K=4~7` 的评估指标
  - `feature_filter`：所有候选症状及其全局发生率、是否保留
  - `patient_clusters`：每位患者的聚类标签
  - `cluster_profiles`：每类中每个症状的频数、频率、全局频率、提升度
- `dendrogram.png`：层次聚类树状图

## 健壮性与可复核性

- 列名、缺失值填补数、剔除症状清单会写入结果文件。
- 若输入列存在非二值异常取值，脚本会抛出带列名的错误。
- 聚类数选择过程完整输出，便于与手工分析记录核对。

## 实现约束

- 使用 Python 脚本实现，便于直接运行和后续复用。
- 使用项目内 `.venv` 安装依赖，避免污染系统环境。
- 以单元测试覆盖核心计算逻辑，再接入 Excel 读写与作图流程。
